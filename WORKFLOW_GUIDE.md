# Piper 机械臂完整训练流程指南

## 📋 完整工作流程概述

本指南详细说明从手眼标定到模型训练和验证的完整流程。

**重要说明**：本系统采用 **Eye-to-Hand（相机固定）** 配置，相机固定在外部，不在机械臂末端。

---

## 第一步：手眼标定 (Eye-to-Hand)

### 准备工作

1. **硬件准备**
   - 将 AprilTag (36h11 系列，尺寸 50mm) 
   - **固定 RealSense 深度相机**在合适位置（不在机械臂上！）
   - 确保机械臂 CAN 接口 (can0) 已启用

2. **软件依赖检查**
   ```bash
   # 确保已安装必要的库
   pip install opencv-python numpy pyrealsense2 pupil-apriltags
   ```

### 执行标定

运行手眼标定程序：

```bash
python calibrate_hand_eye.py
```

### 标定步骤说明

1. **程序启动后**，等待相机和机械臂初始化完成
2. **移动机械臂**到不同的位置（至少 5-10 个点）：
   - 覆盖工作空间的不同区域
   - 确保 AprilTag 始终在相机视野内
   - 尝试不同的高度和角度
3. **在每个位置**：
   - 确保 AprilTag 清晰可见
   - 按 `c` 键采集当前样本点
4. **采集足够样本后**，按 `s` 键计算并保存标定结果
5. **按 `q` 键**退出程序

### 标定原理 (Eye-to-Hand)

由于相机是固定的，我们采用简化的方法：
- 在机械臂末端贴 AprilTag
- 移动机械臂到多个位置
- 同时记录机械臂末端位置（机械臂坐标系）和 AprilTag 位置（相机坐标系）
- 计算两个坐标系之间的平均偏移量

### 标定结果

程序会生成 `simple_hand_eye.json` 文件，包含：
- `camera_to_robot_offset`: 相机到机械臂的坐标偏移 (X, Y, Z)，单位：米
- `camera_intrinsics`: 相机内参

## 第二步：手动数据收集

### 准备工作

1. **确保已完成手眼标定**，`simple_hand_eye.json` 文件存在
2. **安装 3D 鼠标驱动**（如使用 SpaceMouse）：
   ```bash
   pip install pyspacemouse
   ```
3. **准备物体和目标位置**：
   - 将另一个 AprilTag 贴在要抓取的物体上
   - 确定目标放置位置

### 执行数据收集

运行数据收集程序：

```bash
python manual_collect.py
```

### 控制说明

| 操作 | 功能 |
|------|------|
| 3D 鼠标移动 | 控制机械臂 X/Y/Z 方向 |
| 3D 鼠标按钮 0 | 夹爪打开 |
| 3D 鼠标按钮 1 | 夹爪关闭 |
| 键盘空格键 | **+10 奖励（最重要！）** |
| 键盘 `s` 键 | 保存数据 |
| 键盘 `q` 键 | 退出程序 |

### 数据收集建议

- **收集数量**：建议收集 5-10 个 episode
- **每 episode 步数**：200 步
- **奖励时机（关键！）**：
  - ✅ 机械臂移动到物体附近时 → 按空格
  - ✅ 成功抓住物体时 → 立即按空格
  - ✅ 把物体放到目标位置时 → 立即按空格
  - ✅ 任何你觉得做得好的时刻 → 按空格
- **操作多样性**：尝试不同的抓取策略和路径
- **及时奖励**：完成动作后**立即**按空格，不要等！

### 输出文件

程序会生成 `spacemouse_data_256x256x9.npz`，包含：
- `observations`: 观测数据 (N, 9, 256, 256)
- `actions`: 动作数据 (N, 4)
- `rewards`: 奖励数据 (N,)
- `next_observations`: 下一时刻观测
- `discounts`: 折扣因子

## 第三步：行为克隆预训练

### 执行预训练

运行预训练程序：

```bash
python train_manual_data.py
```

### 预训练配置

程序会提示输入：
1. **数据文件路径**：默认 `spacemouse_data_256x256x9.npz`
2. **训练 Epochs**：默认 100（建议 100-200）
3. **Batch Size**：默认 32

### 预训练过程

- 程序会加载手动收集的数据
- 使用行为克隆（Behavior Cloning）方法预训练 Mentor 网络
- 网络学习专家的操作方式
- 每 20 个 Epoch 保存一次中间模型
- 保存最佳模型为 `pretrained_snapshot_256x256x9.pt`

### 输出文件

- `pretrained_snapshot_256x256x9.pt`：最佳预训练模型
- `pretrained_epoch_XX_256x256x9.pt`：中间检查点

## 第四步：强化学习训练

### 配置检查

确保 `piper/cfgs/config.yaml` 中的配置正确：

```yaml
# 机械臂设置
use_sim: false          # 使用真实机械臂
use_apriltag: true      # 使用 AprilTag 检测
visualize: true         # 显示摄像头画面

# 预训练模型路径（会自动加载）
snapshot_path: 'pretrained_snapshot_256x256x9.pt'

# 训练参数
device: cuda:0          # 训练设备
num_train_frames: 2100000  # 总训练帧数
eval_only: false        # 训练模式（不是仅评估）
```

### 执行训练

确保预训练模型文件在工作目录下，然后运行：

```bash
python train_piper.py
```

### 训练过程

1. **初始化阶段**
   - 程序会自动检测并加载预训练模型
   - 初始化机械臂、相机和 AprilTag 检测器
   - 初始化经验回放缓冲区

2. **训练阶段**
   - 使用预训练权重作为起点
   - 通过强化学习继续优化策略
   - 每 10000 帧进行一次评估
   - 每 10000 步保存一次模型快照

3. **手动奖励**
   - 训练过程中仍可按空格键给予手动奖励
   - 奖励会立即影响当前 episode
   - 如果看到好的动作，及时给奖励！

### 输出文件

训练过程中会在 `exp_local/` 目录下生成：
- `snapshots/`: 模型快照
- `videos/`: 评估视频
- `tb/`: TensorBoard 日志

---

## 第五步：模型验证

### 使用训练好的模型进行验证

修改 `piper/cfgs/config.yaml`：

```yaml
eval_only: true          # 仅评估模式
snapshot_path: 'path/to/your/snapshot.pt'  # 要验证的模型路径
```

然后运行：

```bash
python train_piper.py
```

### 验证过程

- 程序会加载指定的模型
- 在评估环境中运行 10 个 episode
- 计算平均成功率和平均奖励
- 保存评估视频

### 成功标准

物体到目标位置的距离 ≤ 0.07 米（7厘米）即为成功。

## 🔧 故障排除

### 问题 1：手眼标定输出全为 0

**原因**：可能是机械臂位置读取失败或样本点不足

**解决方法**：
1. 检查机械臂 CAN 接口是否正常
2. 确保采集了至少 3 个样本点
3. 查看终端输出的错误信息

### 问题 2：找不到 `piper_sdk`

**解决方法**：
1. 确保 Piper 机械臂 SDK 已正确安装
2. 检查 Python 路径是否包含 SDK 目录
3. 确认 `can0` 接口是否可用

### 问题 3：段错误 (Segmentation Fault)

**解决方法**：
1. 使用 `manual_collect.py` 而不是 `manual_train.py`
2. 检查是否有硬件访问冲突
3. 确保所有依赖库版本兼容

### 问题 4：预训练模型无法加载

**解决方法**：
1. 检查 `pretrained_snapshot_256x256x9.pt` 文件是否存在
2. 确认 `config.yaml` 中的 `snapshot_path` 路径正确
3. 检查模型文件是否损坏

### 问题 5：AprilTag 检测不到

**解决方法**：
1. 确保 AprilTag 尺寸设置正确 (`tag_size: 0.05`)
2. 调整光照条件，避免过曝或过暗
3. 确保 AprilTag 完全在相机视野内
4. 检查 `camera_calibration.npz` 是否存在

### 问题 6：坐标转换不正确

**原因**：可能是手眼标定的偏移量计算有误

**解决方法**：
1. 重新进行手眼标定
2. 确保采集足够多的样本点（5-10个或更多）
3. 确保样本覆盖工作空间的不同区域
4. 检查 `simple_hand_eye.json` 中的偏移值是否合理

## 💡 使用技巧

### 1. 手眼标定 (Eye-to-Hand)
- 采样点应覆盖工作空间的不同区域
- 避免采样点过于集中
- 确保每个采样点都能稳定检测到 AprilTag
- 采样时机械臂保持静止，不要在运动中采样

### 2. 数据收集
- 保持操作流畅，避免剧烈运动
- 在关键动作点及时给予奖励（立即按空格！）
- 尝试不同的策略，增加数据多样性
- 如果一个 episode 做得不好，可以提前退出重新来

### 3. 预训练
- 监控 Loss 曲线，确保训练收敛
- 如果 Loss 不下降，检查数据质量
- 适当增加 Epochs 数量（100-200）
- Loss 降到 0.1 以下通常就算不错了

### 4. 强化学习训练
- 从预训练模型开始可以显著加速收敛
- 监控成功率和奖励曲线
- 如果训练不稳定，降低学习率
- 训练过程中仍可按空格给手动奖励

### 5. 验证
- 在不同条件下多次验证
- 分析失败案例，改进数据收集
- 对比不同版本的模型性能
- 成功率超过 80% 就算比较好了

## 📊 观测和动作空间

### 观测空间
- **形状**: (9, 256, 256)
- **说明**: 3 帧 RGB 图像堆叠，每帧 3 通道，捕捉运动信息
- **数据类型**: uint8 (0-255)

### 动作空间
- **形状**: (4,)
- **说明**: 
  - [0]: X 轴移动 (-1 到 1)
  - [1]: Y 轴移动 (-1 到 1)
  - [2]: Z 轴移动 (-1 到 1)
  - [3]: 夹爪控制 (-1 到 1，-1=关闭，1=打开)
- **数据类型**: float32
